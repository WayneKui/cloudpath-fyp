Authentication

All API v1 endpoints require an API key in the Authorization header: 
Authorization: Bearer cpk_a1b2c3d4e5f6...
Keys are tied to your account and inherit your Max-tier permissions. Revoked keys reject with 401.

Endpoints

GET /api/v1/whoami

Verify the key is valid. Returns your user id, email, and tier.

GET /api/v1/scans?limit=20

Past scan runs, newest first. limit up to 100.

GET /api/v1/scans/latest

Convenience — the most recent scan with KPIs.

POST /api/v1/scans

Trigger a new full-pipeline scan. Returns 202 with job_id and a poll_url.

GET /api/v1/attack-paths

Current attack paths from your latest ingested state. Same data as the Scanner graph.

GET /api/v1/exports/paths.csv and /api/v1/exports/paths.json

Downloadable exports of the current attack paths.

Webhook payloads

Each event delivers a JSON envelope like this: 
{
  "event": "scan.completed",
  "event_id": "e2fef8f4-9c1b-4c...",
  "created_at": 1720000000,
  "data": {
    "scan_type": "manual",
    "total_paths": 3,
    "critical": 0,
    "high": 1,
    "medium": 2,
    "low": 0,
    "paths_preview": [ { "title": "…", "severity": "High", "score": 7.4 } ]
  }
}
Verify the signature with: 
expected = "sha256=" + hmac_sha256(secret, raw_body).hex()
constant_time_equal(expected, request.headers["X-CloudPath-Signature"])
